## Why longevity matters

Software rarely fails because it cannot ship. It fails when it cannot evolve.

> Favor systems that are clear over systems that are merely clever.

## A practical framework

1. Start with explicit constraints.
2. Design boundaries around change.
3. Document decisions where they happen.

## Example policy

```ts
export const maintainability = {
  tests: 'required',
  docs: 'close to code',
  ownership: 'clear',
}
```

## Final thought

Long-lived systems are authored with care, not speed alone.
